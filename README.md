# Liquibase POC — Spring Boot + PostgreSQL

A proof of concept showing how to integrate Liquibase into an **existing** Spring Boot project
that already has tables and data — without breaking anything.

---

## Prerequisites

| Software | Version | Install |
|---|---|---|
| Java | 17+ | [adoptium.net](https://adoptium.net) or `choco install temurin17` |
| Docker Desktop | Latest | [docker.com](https://www.docker.com/products/docker-desktop) |
| Git Bash | Latest | Bundled with [Git for Windows](https://git-scm.com/downloads) |
| Liquibase CLI | Latest | `choco install liquibase` |

Verify everything is working:
```bash
java -version
docker -v
liquibase --version
```

## Project Structure

```
liquibase-poc/
├── compose.yaml                         # starts postgres with existing tables
├── init.sql                             # pre-existing table (before Liquibase)
├── scripts/
│   └── base.sh                          # generates baseline changelog from existing DB
└── src/main/resources/
    ├── application.properties
    └── db/changelog/
        ├── master.xml                   <- single entry point, includes all changelogs
        └── ddl/
            └── 001-baseline-existing-tables.xml   <- generated by base.sh
```

## Existing Database (Before Liquibase)

The `init.sql` contains one pre-existing table that was created before Liquibase existed:

```sql
CREATE TABLE users (
    id       SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email    VARCHAR(100)
);
```

The goal is to let Liquibase take over schema management **without dropping or recreating this table**.

## How to Run

Follow these steps **in order**.

---

### Step 1 — Start the Database

Start PostgreSQL via Docker. This runs `init.sql` automatically, creating
the `users` table — simulating a real DB that existed before Liquibase.

```bash
docker compose up -d
```

> **No Docker?** Apply `init.sql` manually:
> ```bash
> psql -U myuser -d mydatabase -f init.sql
> ```

Verify the table exists but Liquibase has not run yet:
```bash
docker exec -it <postgres-container-name> psql -U myuser -d mydatabase -c "\dt"
```

You should see `users` — but **no** `DATABASECHANGELOG` table yet.
That proves Liquibase hasn't touched anything.

---

### Step 2 — Run the App (confirm existing table is intact)

Start the app to confirm it connects to the DB successfully.

```bash
./gradlew bootRun
```

> At this point `master.xml` has no changelogs included yet so Liquibase
> has nothing to run. That is expected — we fix that in the next steps.

---

### Step 3 — Generate the Baseline

Run the script to snapshot the existing `users` table into a changelog file:

```bash
bash scripts/base.sh
```

This does two things:

**1. Generates** `src/main/resources/db/changelog/ddl/001-baseline-existing-tables.xml`

A snapshot of the `users` table. It uses `<preConditions onFail="MARK_RAN">` so when
Spring runs Liquibase, it will:
- See `users` already exists and skip creation
- Still record the changeset in `DATABASECHANGELOG` as `MARK_RAN`
- Never drop or recreate the table or its data

**2. Proves** the snapshot is 1-to-1 with the live DB

The script runs a `diff` between the generated file and the actual DB.
Output should include:
```
No unexpected differences found
```
This confirms the changelog is an exact match of what is in the DB.

---

### Step 4 — Add Baseline to master.xml

Open `src/main/resources/db/changelog/master.xml` and add under Phase 0:

```xml
<!-- PHASE 0: BASELINE — tables already created by init.sql -->
<include file="db/changelog/ddl/001-baseline-existing-tables.xml"
         relativeToChangelogFile="false"/>
```

---

### Step 5 — Run the App Again (Liquibase takes over)

```bash
./gradlew bootRun
```

On startup Liquibase will read `master.xml`, find the `users` table already exists,
and record it as `MARK_RAN` — no data loss, no recreation.

---

### Step 6 — Add New Changes (Newcomer Demo)

This shows how any team member adds a new change going forward —
no manual SQL, no "did you run the script?" — just a changeset file.

Create `src/main/resources/db/changelog/ddl/002-add-profile-table.xml`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<databaseChangeLog xmlns="http://www.liquibase.org/xml/ns/dbchangelog"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.liquibase.org/xml/ns/dbchangelog
        https://www.liquibase.org/xml/ns/dbchangelog/dbchangelog-4.20.xsd">

    <!-- Add a column to the existing users table -->
    <changeSet id="ddl-002" author="dave.newcomer">
        <addColumn tableName="users">
            <column name="full_name" type="VARCHAR(150)"/>
        </addColumn>
        <rollback>
            <dropColumn tableName="users" columnName="full_name"/>
        </rollback>
    </changeSet>

    <!-- Add a brand new table -->
    <changeSet id="ddl-003" author="dave.newcomer">
        <createTable tableName="profiles">
            <column name="id" type="BIGSERIAL">
                <constraints primaryKey="true" nullable="false"/>
            </column>
            <column name="user_id" type="INT">
                <constraints nullable="false" foreignKeyName="fk_profile_user"
                             references="users(id)"/>
            </column>
            <column name="bio" type="TEXT"/>
        </createTable>
        <rollback>
            <dropTable tableName="profiles"/>
        </rollback>
    </changeSet>

</databaseChangeLog>
```

Add it to `master.xml` under Phase 1:
```xml
<!-- PHASE 1: DDL — new structure added after Liquibase -->
<include file="db/changelog/ddl/002-add-profile-table.xml"
         relativeToChangelogFile="false"/>
```

Run the app:
```bash
./gradlew bootRun
```

- `full_name` column appears on `users` — existing rows unaffected
- `profiles` table is created
- `users` table and its data are untouched
- `DATABASECHANGELOG` shows `EXECTYPE = EXECUTED` for Dave's changesets

## Troubleshooting

**`Unable to determine Java version` in base.sh**
Make sure you are running in Git Bash, not CMD or PowerShell.

**`Cannot find database driver` in base.sh**
Run `./gradlew dependencies` first to download the PostgreSQL jar into Gradle cache.
The script finds it automatically from `~/.gradle`.

**`liquibase: command not found`**
Install via Chocolatey: `choco install liquibase`, then reopen Git Bash.

**Table dropped or data lost on bootRun**
- Confirm `spring.jpa.hibernate.ddl-auto=none` in `application.properties`
- Confirm Step 3 and Step 4 were done before Step 5
- Check that `001-baseline-existing-tables.xml` is included in `master.xml`
